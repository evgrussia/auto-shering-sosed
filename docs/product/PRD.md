# Product Requirements Document (PRD)

## 1. Обзор

**Продукт:** P2P-каршеринг между соседями в формате Telegram WebApp.  
**MVP:** бронирование, передача ключей вручную, базовая верификация через Госуслуги, листинг авто, рейтинги, задел под арбитраж.  
**Ссылки:** [Vision](vision.md), [Competitor Analysis](../research/competitor-analysis.md), [Project Brief](../../context/project-brief.yaml).

## 2. Цели и метрики

| Цель | Метрика |
|------|---------|
| Запуск в нескольких городах | Кол-во городов, кол-во активных листингов |
| Рост доверия | % верифицированных через Госуслуги, средний рейтинг |
| Использование | Кол-во бронирований/мес, повторные бронирования |
| Монетизация | Комиссия 20–25% с аренды; выручка платформы |

## 3. Пользователи и роли

- **Владелец (собственник авто):** добавляет авто, задаёт цену и правила, принимает/отклоняет заявки, передаёт ключи, получает оплату (post-MVP), ставит отзыв, участвует в арбитраже при споре.
- **Арендатор:** ищет авто, проходит верификацию, бронирует, забирает/возвращает ключи, оплачивает (post-MVP), оставляет отзыв, может инициировать претензию.
- **Платформа (админ/поддержка):** модерация листингов, разрешение споров (арбитраж), настройка интеграций.

## 5. Нефункциональные требования

- **Платформа:** Telegram WebApp (фронт + backend API); данные в РФ; соответствие 152-ФЗ.
- **Госуслуги:** только разрешённые методы проверки документов; хранение только необходимых данных.
- **Доступность:** работа в основных браузерах, поддерживаемых Telegram WebApp.
- **Безопасность:** токены/ключи не в коде; секреты в окружении.

## 6. Ограничения и допущения

- MVP без приёма платежей на платформе: оплата между сторонами (или ручной учёт); ЮKassa — в post-MVP.
- Умные замки и КАСКО — слоты в UX и контрактах; конкретные партнёры — после исследования и договорённостей.
- Верификация Госуслуги — только проверка документов (без кредитной истории и штрафов по API).

## 7. Зависимости

- Доступ к API Госуслуг (проверка документов).
- Telegram Bot + WebApp (BotFather, HTTPS).
- (Post-MVP) ЮKassa: магазин, webhooks, чеки.
- (Post-MVP) Вендор умных замков: API/документация.

## 8. Критерии приёмки MVP

- Владелец может зарегистрироваться, пройти верификацию Госуслуги и опубликовать листинг авто в выбранном городе.
- Арендатор может найти авто по городу/датам, пройти верификацию и отправить заявку на бронирование.
- Владелец может подтвердить или отклонить заявку; после завершения аренды обе стороны оставляют рейтинг и отзыв.
- Передача ключей происходит офлайн по договорённости; в системе фиксируются статусы бронирования.
- Есть задел для прикрепления фото/описания к бронированию для будущего арбитража.
