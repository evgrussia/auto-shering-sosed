# Unit-экономика

## 1. Модель комиссии

- **Комиссия платформы:** 20–25% от суммы аренды (удерживается при оплате через ЮKassa).
- **Пример:** аренда 3000 ₽/сутки → платформа 600–750 ₽; владелец 2250–2400 ₽.
- **Сравнение:** RentRide 15%; целевая цена для клиента на 30–50% ниже коммерческого каршеринга сохраняет ценность при 20–25%.

## 2. Метрики на сделку

| Показатель | Формула / описание |
|------------|---------------------|
| **ARPU (арендатор)** | Средняя сумма аренды × доля бронирований с оплатой через платформу |
| **Комиссия с бронирования** | Сумма аренды × (20–25%) |
| **CAC (привлечение)** | Расходы на привлечение / новые верифицированные пользователи (по кампаниям) |
| **LTV (упрощённо)** | Средний чек × среднее кол-во бронирований в год × комиссия; или по когортам |

## 3. Целевые ориентиры (после запуска платежей)

- **LTV/CAC:** > 3 (целевой порог для здоровой unit-экономики).
- **Повторные бронирования:** рост доли пользователей с ≥2 завершёнными арендами.
- **Доля комиссии в GMV:** 20–25%; после вычета комиссии платёжной системы и операционных расходов — положительная маржинальность на сделку.

## 4. Издержки на сделку

- Комиссия платёжной системы (ЮKassa): порядка 2–3% + фикс за транзакцию.
- Поддержка и арбитраж: оценка времени на спор и модерацию; масштабирование при росте.
- Инфраструктура и разработка: фикс + масштабирование с объёмом.

## 5. Чувствительность

- При снижении комиссии до 15% (как RentRide) маржа падает; необходимо компенсировать объёмом или снижением CAC/операционных расходов.
- Рост доли повторных бронирований снижает зависимость от CAC и повышает LTV.
