# План метрик и трекинг

## 1. Северная звезда и ключевые метрики

| Метрика | Описание | Цель |
|---------|----------|------|
| **GMV (объём аренд)** | Сумма всех оплаченных аренд (руб) за период | Рост по месяцам |
| **Выручка платформы** | Комиссия 20–25% от GMV | Рост, маржинальность |
| **Активные бронирования** | Кол-во бронирований со статусом «Завершено» за период | Рост |
| **Повторные бронирования** | % бронирований от пользователей с ≥2 завершёнными арендами | Рост удержания |

## 2. Воронка и события

### Регистрация и верификация

| Событие | Описание |
|---------|----------|
| `user_signed_up` | Регистрация через Telegram |
| `verification_started` | Начало верификации Госуслуги |
| `verification_completed` | Успешная верификация |
| `verification_failed` | Ошибка или отказ верификации |

### Владелец

| Событие | Описание |
|---------|----------|
| `listing_created` | Создан листинг авто |
| `listing_updated` | Изменение листинга |
| `listing_deactivated` | Листинг скрыт/удалён |
| `booking_request_received` | Получена заявка на бронирование |
| `booking_confirmed_by_owner` | Владелец подтвердил заявку |
| `booking_rejected_by_owner` | Владелец отклонил заявку |
| `rental_completed` | Аренда завершена (владелец отметил) |
| `review_submitted` | Владелец оставил отзыв |

### Арендатор

| Событие | Описание |
|---------|----------|
| `search_performed` | Поиск по городу/датам (опционально: фильтры) |
| `listing_viewed` | Просмотр карточки листинга |
| `booking_request_sent` | Отправлена заявка на бронирование |
| `booking_cancelled_by_renter` | Арендатор отменил заявку/бронирование |
| `rental_completed` | Аренда завершена |
| `review_submitted` | Арендатор оставил отзыв |

### Оплата (post-MVP)

| Событие | Описание |
|---------|----------|
| `payment_initiated` | Инициирована оплата (аренда/депозит) |
| `payment_completed` | Оплата успешна |
| `payment_failed` | Ошибка оплаты |
| `refund_initiated` | Инициирован возврат |
| `refund_completed` | Возврат выполнен |

### Арбитраж (post-MVP)

| Событие | Описание |
|---------|----------|
| `dispute_created` | Создана претензия |
| `dispute_resolved` | Претензия разрешена (исход: в пользу владельца/арендатора/раздельно) |

## 3. Агрегаты и отчёты

- **По городам:** кол-во листингов, кол-во бронирований, GMV, конверсия поиск → заявка → подтверждение → завершение.
- **По когортам:** регистрация по неделям/месяцам; retention (повторное бронирование в течение 30/90 дней).
- **Качество:** средний рейтинг владельцев и арендаторов; кол-во споров и время разрешения (post-MVP).
- **Unit-экономика:** см. [unit-economics.md](unit-economics.md).

## 4. Инструменты и хранение

- События отправлять в backend; хранить в БД и/или передавать в аналитический стек (например, Amplitude, Mixpanel, или self-hosted).
- Дашборды: воронка, GMV, выручка, активные пользователи, повторные бронирования.
- Соблюдение 152-ФЗ: минимизация ПД в событиях; анонимизация где возможно.
