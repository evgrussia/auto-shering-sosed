# Вендоры умных замков (предварительное исследование)

Цель: интеграция с готовыми решениями для открытия авто через приложение (post-MVP).

## 1. Smartcar (международный)

- **Тип:** API/SDK для lock/unlock и данных автомобиля.
- **Охват:** 39+ брендов, авто с ~2015 г.
- **Интеграция:** OAuth (Smartcar Connect), токены доступа; SDK: Node.js, Python, Java, Go, Ruby.
- **Ограничения:** ориентирован на США/западные рынки; необходимо уточнять поддержку РФ и локальных марок.

## 2. Направления для РФ

- **ОЕМ-решения:** производители авто с поддержкой цифрового ключа (приложение производителя или партнёрский API) — выборочно доступны в РФ.
- **Аппаратные решения:** ключ-боксы (lockbox) на машину, открытие по коду/приложению; поставщики для car-sharing и rental в РФ — отдельный поиск (локальные интеграторы, телематика).
- **Телематика:** российские провайдеры телематики и «умного авто» — возможность блокировки/разблокировки по API при наличии оборудования у владельца.

## 3. Рекомендации для продукта

1. **Post-MVP:** заложить в архитектуру слот «умный замок»: абстракция (открыть/закрыть по `booking_id`), конкретный провайдер подставляется позже.
2. **Исследование:** провести отдельный запрос по российским вендорам (телематика, ключ-боксы, интеграторы каршеринга) и по возможности — по условиям OEM для массовых марок в РФ.
3. **MVP:** без умных замков; передача ключей вручную; в PRD и сценариях зафиксировать переход на «открытие через приложение» при наличии интеграции.
